# 中文在线短剧 Agent RAG 检索提升 Checklist

## 文档完整性

- [ ] 方案说明了当前 RAG 链路：结构化资产 -> embedding -> Milvus -> PostgreSQL 回表。
- [ ] 方案明确指出当前是单路向量召回。
- [ ] 方案覆盖角色、场景、变体、图片视觉语义、分镜语义。
- [ ] 方案包含网上调研来源链接。

## 评测闭环

- [ ] 已定义至少 50 条评测 query。
- [ ] 每条 query 有期望命中资产或分镜 ID。
- [ ] 评测 query 覆盖精确名称、自然语言、风格描述、变体、场景、分镜、负样本。
- [ ] 能计算 `Hit@K`、`Recall@K`、`MRR`、`Precision@K`。

## 召回能力

- [ ] 向量召回支持内部 topN 大于最终返回 limit。
- [ ] 精确名称匹配可以把目标资产召回进候选池。
- [ ] 标签、分类、项目、资产类型可以参与候选召回或过滤。
- [ ] 多路召回结果可以统一为候选结构。

## 融合与重排

- [ ] 多路召回使用 RRF 或等价融合策略，而不是直接相加不可比 score。
- [ ] 规则重排覆盖名称命中、类型匹配、标签命中、图片可用、审核状态。
- [ ] 同一主体的多个变体不会无控制地占满首屏。
- [ ] 模型重排可解释，能输出重排原因或 debug 信息。

## 多粒度向量

- [ ] 资产主体向量和资产变体向量边界清晰。
- [ ] 图片视觉语义来自视觉模型生成的文本，而不是图片 URL。
- [ ] 分镜向量使用独立 collection，不污染资产主体 collection。
- [ ] metadata 中保留 `source_table`、`source_id`、项目、类型等追溯字段。

## 端到端场景

- [ ] 输入“叶逍遥”时，目标角色进入 top 1。
- [ ] 输入“清冷古风白衣男性角色”时，相关角色进入 top 5。
- [ ] 输入“山中道台、云雾、古风场景”时，相关场景进入 top 5。
- [ ] 输入“心魔版角色”时，能命中变体而不是只返回主体。
- [ ] 输入“蓝白长袍、高马尾、仙侠气质”时，视觉语义链路能召回相关图或资产。
- [ ] 输入一段台词或镜头描述时，能召回分镜记录，并能扩展关联资产。

## 回归检查

- [ ] 原有 `/api/v1/asset-search/semantic` 的 `query + limit` 调用保持兼容。
- [ ] 旧数据未补视觉语义时，检索仍可正常返回资产主体。
- [ ] Milvus 不可用时，接口能给出可诊断错误日志。
- [ ] PostgreSQL 回表失败时，不返回无来源的脏结果。
- [ ] 检索日志不泄露 `.env`、API key、数据库密码等敏感信息。
