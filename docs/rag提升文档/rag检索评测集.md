# RAG 检索评测集

> 这份表用于人工评估中文在线短剧 Agent 的资产 RAG 检索效果。  
> 它不是接口文档，而是“检索质量验收表”：每一行都是一个用户可能输入的 query，并明确写出应该命中哪个资产、应该排第几、实际跑出来是什么。

## 使用方式

1. 调用检索接口：

```http
POST /api/v1/asset-search/semantic
```

请求示例：

```json
{
  "query": "叶逍遥",
  "limit": 5
}
```

2. 看返回的 `items`。
3. 把目标资产实际排第几、实际分数、实际命中资产名填到表里。
4. 按“期望排名”判断是否通过。
5. 失败时，在“问题判断”里写清楚原因，比如：

- 未召回
- 排名太低
- 返回了错误资产类型
- 同名资产干扰
- 语义字段缺失
- 向量文本太短
- 变体和主体混在一起
- 数据库混入非短剧资产

## 通过标准

| 类型 | 期望 |
|---|---|
| 精确名称检索 | 目标资产排第 1 名 |
| 别名/括号名检索 | 目标资产进入前 3 名 |
| 语义描述检索 | 目标资产进入前 5 名 |
| 场景描述检索 | 目标场景进入前 5 名 |
| 变体检索 | 优先命中变体；当前接口如果只返回主体，主体进入前 5 名暂算通过 |
| 负样本 | 不应高置信返回无关短剧资产 |

## 这份评测集怎么指导后续优化

### 如果精确名称失败

优先做：

1. 名称精确匹配召回。
2. `name / display_name / parent_entity_name` 加权。
3. RRF 融合时让 exact match 稳定进入候选池。

### 如果语义描述失败

优先做：

1. 检查 `vector_text` 是否缺少对应字段。
2. 扩大 Milvus 召回 topN。
3. 增加规则重排和 rerank。

### 如果场景和角色混淆

优先做：

1. 增加 `asset_kind` 过滤。
2. 查询理解中识别“角色/人物/场景/地点/会所/医院”等类型词。
3. 重排时惩罚资产类型不匹配的结果。

### 如果变体检索失败

优先做：

1. 变体向量独立召回。
2. 主体和变体结果合并展示。
3. 对 `variant.name / usage_context / visual_prompt` 加权。

### 如果负样本返回很自信

优先做：

1. 增加最低分阈值。
2. 增加“无明确匹配”的响应策略。
3. 避免把非短剧资产混入默认检索结果。
