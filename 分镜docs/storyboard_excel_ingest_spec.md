# 分镜 Excel 统一入库 Spec

## 背景

当前系统已经支持通过同一个 Excel 上传接口完成资产入库，主要处理人物、角色、场景等资产数据，并将资产写入结构化数据库和资产向量库。

实际业务中，用户上传的往往是同一份 Excel，里面可能同时包含人物、场景、分镜等多类数据。因此入库能力不应拆成多个入口，而应该在同一个 Excel 入库接口中自动识别不同数据类型，并分别写入对应的数据表和向量库。

本阶段需要先打通分镜入库的最小闭环：上传同一份 Excel 后，人物和场景继续走现有资产入库流程，分镜数据新增写入 `common.project_director_storyboard_prompts`，并同步到单独的分镜向量库。

## 目标

- 保持现有 Excel 上传接口不变。
- 上传一份 Excel 后，后端自动识别资产表和分镜表。
- 人物、角色、场景等资产继续走现有资产入库流程。
- 分镜数据写入 `common.project_director_storyboard_prompts`。
- 分镜数据同步到独立的分镜向量库。
- 入库成功后继续删除上传的 Excel 源文件。
- 第一版先保证“同一份 Excel 可以同时入资产和分镜”，为后续分镜检索、资产匹配和生成链路打基础。

## 功能需求

- F1: 入库入口保持不变，继续使用现有 Excel 上传接口：
  `POST /api/v1/asset-ingest/excel/upload`

- F2: 后端接收到一份 Excel 后，需要自动识别里面的数据类型：
  - 人物 / 角色表
  - 场景表
  - 分镜表

- F3: 人物、角色、场景等资产数据继续走现有资产入库流程：
  - 写入 `common.asset_entities`
  - 写入 `common.asset_media`
  - 同步资产向量库 `asset_entity_vectors`

- F4: 分镜数据新增入库流程，写入：
  `common.project_director_storyboard_prompts`

- F5: 分镜行需要至少保留这些业务信息：
  - 项目
  - 集数
  - 镜号
  - 画面描述
  - 镜头运动 / 景别视角
  - 台词
  - 场景
  - 音效
  - 参考画面 / 静态帧信息
  - 原始 Excel sheet 名
  - 原始 Excel 行号

- F6: 同一份 Excel 入库完成后，接口响应需要能看出：
  - 资产入库数量
  - 分镜入库数量
  - 总处理数量
  - 源 Excel 是否已删除

- F7: 如果 Excel 里没有分镜表，不影响原来的资产入库。如果 Excel 里没有资产表，也允许只入分镜。

- F8: 分镜入库后需要同步到向量库，但不复用资产向量 collection。分镜向量单独写入 `project_storyboard_vectors`，用于后续分镜语义检索、分镜召回和根据分镜匹配资产。

## 非功能需求

- N1: 保持现有资产入库接口兼容，不要求对接方更换接口。

- N2: 入库流程需要允许定位不同类型数据的失败原因。例如资产入库成功、分镜入库失败时，需要能从日志或响应里定位是哪一类数据失败。

- N3: 分镜向量库和资产向量库隔离。资产继续写入 `asset_entity_vectors`，分镜写入 `project_storyboard_vectors`。

- N4: 分镜向量文本要尽量贴近检索语义。优先拼接：项目、集数、镜号、画面描述、景别视角、镜头运动、台词、场景、音效、参考画面说明。

- N5: 入库成功后继续删除上传的 Excel 源文件，不长期保留。

- N6: 第一版优先保证稳定入库，不强依赖视觉模型。Excel 里的图片或静态帧可以先记录引用信息，视觉理解后续再补。

## 不做的事

- 第一版不做图片视觉模型理解。
- 第一版不做分镜图自动生成文本描述。
- 第一版不做分镜自动匹配资产。
- 第一版不做视频生成记录写入。
- 第一版不做审核意见流转。
- 第一版不做复杂前端改造，前端只需要能继续调用现有上传接口即可。

## 验收标准

- AC1: 使用现有 Excel 上传接口上传只包含人物或场景的 Excel 时，原有资产入库能力不受影响。

- AC2: 使用现有 Excel 上传接口上传包含分镜 sheet 的 Excel 时，系统能够把分镜行写入 `common.project_director_storyboard_prompts`。

- AC3: 使用同一份同时包含人物、场景、分镜的 Excel 入库时，系统能够分别完成资产入库和分镜入库。

- AC4: 分镜入库后，能在分镜向量库 `project_storyboard_vectors` 中检索到对应分镜文本。

- AC5: 接口响应能体现资产入库数量、分镜入库数量、总处理数量和源 Excel 删除状态。

- AC6: Excel 中没有分镜数据时，接口仍能正常完成资产入库；Excel 中没有资产数据时，接口仍能正常完成分镜入库。

- AC7: 入库成功后，上传到运行目录的 Excel 源文件会被删除。
